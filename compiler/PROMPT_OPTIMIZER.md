# BCube Gold Prompt Optimizer™

## Purpose

The optimizer converts the expanded Prompt IR into a concise, deterministic and image-model-ready production prompt without removing required detail.

## Non-negotiable rule

Optimization MUST NOT replace explicit instructions with rule IDs, module names, shorthand references or unresolved placeholders.

## Optimization sequence

1. Normalize terminology.
2. Remove exact duplicate instructions.
3. Merge compatible repeated constraints.
4. Detect contradictory placements, dimensions, colours and character states.
5. Apply conflict precedence.
6. Verify mandatory section order.
7. Re-inject any mandatory requirement lost during normalization.
8. Run self-contained prompt validation.
9. Emit the final Gold Production Prompt™.

## Required section order

1. Technical and publishing specification
2. Age and developmental style
3. Exact page identity and visible text
4. Layout and protected zones
5. Main scene and visual hierarchy
6. Teacher-child interaction
7. Character definitions and poses
8. Child activity and response space
9. Parent or home connection where applicable
10. Branding and footer requirements
11. Negative constraints
12. Final publication-quality instruction

## Duplicate handling

- Exact duplicates are removed.
- Similar requirements are merged only when meaning is preserved.
- Character identity details are never shortened below the canonical minimum.
- Exact visible text is never paraphrased.
- Safety and negative constraints may repeat when needed to protect critical output.

## Ambiguity handling

The optimizer MUST reject vague phrases such as:

- "use the standard style"
- "follow the common rules"
- "same as previous page"
- "use approved mascot"
- "add suitable content"

Each must be expanded into concrete visual and production instructions.

## Failure conditions

Optimization fails when any of the following remains:

- unresolved variable
- rule or module reference
- contradictory geometry
- conflicting character placement
- missing exact text
- missing activity response area
- missing negative constraints
- ambiguous visual instruction

## Output

The output is one complete, self-contained prompt that can be sent directly to the image generation model without external context.
