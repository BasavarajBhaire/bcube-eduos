# BCube Evidence-Based Render QA

A production render cannot pass by declaring visual booleans such as `official_star_hash_match: true`.
The validator independently verifies evidence.

## Required evidence

1. **Artifact evidence**
   - PNG format
   - 2480 × 3508 pixels
   - 300 DPI metadata

2. **Component evidence generated by the compositor**
   - component name
   - approved source path
   - approved SHA-256
   - actual bounds
   - locked template bounds
   - explicit overlap allowance only where intentional

3. **Repository-owned book text policy**
   - the render manifest provides only detected text and detector identity
   - required and prohibited terms come from `book-text-policies.json`
   - cross-book contamination fails automatically

4. **Human approval bound to the artifact**
   - reviewer
   - ISO approval date
   - status `APPROVED`
   - exact rendered PNG SHA-256

## Failure examples

A Confidence Builders cover fails when detected text includes:

- `GOOD COMMUNICATION MAKES FRIENDS`
- `COMMUNICATION CHAMPIONS`
- `LISTEN SPEAK SHARE`

A page also fails when:

- an official asset source hash differs from its approved hash;
- component bounds differ from the locked template;
- required components are missing;
- prohibited components are present;
- non-permitted components overlap;
- approval refers to a different artifact hash;
- full-page AI generation is declared or used.

## Acceptance

A page is production-eligible only at:

- score: **100/100**
- critical defects: **0**
- machine QA: **PASS**
- human approval: **APPROVED** and hash-bound
