# BCube Page Composer™

BCube Page Composer separates AI-generated illustration artwork from deterministic publishing layout.

## Official pipeline

1. Stage 1 — Prompt Compiler
2. Stage 2 — Illustration Generator (artwork only; no logo, title, page number, or guidance text)
3. Stage 3 — BCube Page Composer™
4. Stage 4 — Visual QA and Human Approval
5. Stage 5 — PDF and Prepress Builder

## Composer responsibilities

- Place the approved BCube logo without redrawing or altering it.
- Place approved Star mascot assets without regenerating the mascot.
- Apply official typography, colors, spacing, safe margins, headers, footers, badges, guidance boxes, and page numbers.
- Combine fixed assets with the generated illustration layer.
- Export deterministic A4 portrait production pages.
- Produce a page manifest containing source hashes and layout metadata.

## Asset policy

AI image generation must not generate or imitate the BCube logo, Star mascot, book title typography, badges, icons, page numbers, teacher guidance, parent guidance, or publisher text. These elements are owned by the Page Composer and must come from approved fixed assets and structured content.

## Initial structure

```text
stage3-page-composer/
├── composer/
│   └── page_composer.py
├── design-system/
│   ├── colors.json
│   ├── typography.json
│   └── layout.json
├── templates/
│   └── interior-page.json
├── assets/
│   └── registry.json
└── output/
```

Binary logo, mascot, badge, icon, and illustration assets are intentionally not generated by this scaffold. Approved files must be added to the paths defined in `assets/registry.json`.
