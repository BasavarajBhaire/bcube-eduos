# Architecture Decision Records

Architecture Decision Records (ADRs) preserve why BCube EduOS works the way it does.

## Status values

- Proposed
- Accepted
- Superseded
- Rejected

## Rules

1. Accepted ADRs are binding.
2. An accepted ADR is never edited to reverse its decision.
3. A changed decision requires a new ADR that supersedes the earlier record.
4. Founder approval is required for brand identity, curriculum authority, acceptance threshold, release immutability and major-version changes.
