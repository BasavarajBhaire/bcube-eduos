# Publishing Engine Rules

This directory contains normative rules owned by the BCube Publishing Engine™.

## Scope

Publishing rules govern page geometry, trim, bleed, safe areas, binding margins, typography production constraints, print readiness, export profiles, branding zones, and protected activity areas.

## Naming

`PE-<CATEGORY>-<NNN>.md`

Examples:
- `PE-GEO-001.md`
- `PE-TYP-001.md`
- `PE-ACT-001.md`

## Rule status

Rules move through: Draft → Review → Approved → Active → Superseded → Deprecated → Archived.

## Required sections

Every rule MUST include purpose, scope, normative requirement, rationale, inputs, outputs, dependencies, validation, failure severity, compiler mapping, examples, anti-patterns, and change history.
