# Educational Engine Rule Index

Owner: Chief Curriculum Architect  
Prefix: `EE-`

## Active foundation rules

- `EE-OBJ-001` — Each page MUST define one measurable primary learning objective.
- `EE-COMP-001` — Each page MUST map to at least one BCube competency.
- `EE-DEV-001` — Content MUST match the target developmental stage.
- `EE-ASSESS-001` — Every activity MUST expose observable assessment evidence.
- `EE-VAL-001` — Educational validation MUST pass before production release.

Outputs: objective, competency, developmental level, vocabulary, assessment evidence, and expected child outcome.
